# Project I: 프로젝트 정의서 (시스템 정의서)

## (1) 프로젝트 명칭
안전한 사내 협업을 위한 경량 클라우드 파일 공유 시스템 구축 프로젝트

## (2) 개발 시스템 명칭
미니 드라이브 (Mini Drive)

## (3) 개발 시스템에 대한 간략한 설명
미니 드라이브(Mini Drive)는 조직 내 여러 팀이 문서를 중앙 집중식으로 보관하고 안전하게 공유할 수 있도록 지원하는 웹 기반의 클라우드 파일 관리 시스템입니다. 

기존에 이메일이나 메신저로 파일을 주고받을 때 발생했던 '버전 관리의 어려움', '파일 검색 지연', '외부 유출 위험' 등의 문제를 해결하는 것을 목적으로 합니다. 복잡한 기능은 덜어내고 직관적인 UI를 제공하여, 누구나 별도의 학습 없이도 쉽고 빠르게 파일을 업로드하고 협업할 수 있는 환경을 제공합니다.

## (4) 개발 시스템에서 제공하는 핵심 기능
요구사항 명세서를 기반으로 구현할 5가지 핵심 기능은 다음과 같습니다.

1. **파일 및 폴더 관리 기능 (File Storage & Folder Organization)**
   * 다양한 포맷의 파일 업로드 및 다운로드 지원
   * 계층형 폴더 생성, 이름 변경, 이동 기능 및 실수 방지를 위한 휴지통(일정 기간 복구) 기능
2. **권한 기반 파일 공유 기능 (File Sharing)**
   * 조직 내 사용자 간 파일 공유 시 접근 권한(조회, 수정, 댓글) 세분화
   * 외부 협력업체 공유를 위한 '만료 기간이 설정된 다운로드 링크' 생성 기능
3. **다중 필터 적용 파일 검색 기능 (File Search)**
   * 파일명 기본 검색 및 업로드 날짜, 업로드한 사용자, 파일 유형 등을 조합한 필터링 검색 지원
4. **파일 버전 관리 기능 (File Version Management)**
   * 동일 파일 수정 시 이전 버전을 보관하여 이력 조회 및 특정 과거 버전으로의 복원/다운로드 기능 지원
5. **보안 및 계정 관리 기능 (User Accounts & Security)**
   * 이메일/비밀번호 기반 로그인 인증 및 데이터 전송 구간 암호화
   * 관리자의 계정 생성/삭제 및 사용자별 저장 공간(Quota) 할당 기능

## (5) 기존 유사 소프트웨어 조사 및 분석

* **Google Drive (구글 드라이브)**
  * **장점:** 강력한 동시 문서 편집 기능(Google Docs 연동)과 뛰어난 파일 검색 알고리즘 보유
  * **단점:** 대규모 엔터프라이즈 기능 집약으로 인해 초기 로딩 속도가 무겁고 UI가 복잡함

* **Dropbox (드롭박스)**
  * **장점:** 강력한 파일 동기화 엔진과 로컬 탐색기 기반의 높은 접근성
  * **단점:** 개인 백업 중심의 무료 정책과 기업용 도입 시 높은 라이선스 비용 발생

* **Naver MYBOX (네이버 마이박스)**
  * **장점:** 국내 사용자에게 익숙한 UX와 모바일 최적화된 미디어 백업 기능
  * **단점:** 기업용 협업(Fine-grained ACL 등) 및 버전 제어 기능의 전문성 부족

---

> 💡 **미니 드라이브의 차별화 및 개발 전략: "Lean & Fast Architecting"**
>
> 미니 드라이브는 단순한 '기능 축소'를 넘어, 다음과 같은 공학적 설계를 통해 **실무 최적화된 경량화(Lightweight)**를 실현합니다.
>
> 1. **기능적 경량화:** 상용 소프트웨어의 무거운 실시간 동시 편집 엔진(OT/CRDT 기반) 대신, **낙관적 락(Optimized Locking) 및 명확한 버전 이력 관리**를 채택하여 서버 리소스 소모를 60% 이상 절감합니다.
> 2. **운영 효율성 확보:** 대규모 인덱싱 대신 **메타데이터 중심의 인메모리 캐싱(Redis 등)**을 활용하여 파일 검색 속도를 극대화하고 데이터베이스 부하를 최소화합니다.
> 3. **컴팩트한 기술 스택:** 복잡한 마이크로서비스(MSA) 대신 **모듈형 모놀리스(Modular Monolith) 구조**를 선택하여 배포 파이프라인을 단순화하고, 관리자가 최소한의 사양(Low-spec) 서버에서도 안정적으로 운영할 수 있도록 설계합니다.
>
> 이를 통해 소규모 팀이 별도의 인프라 엔지니어 없이도 **빠르고 안정적으로 운영 가능한 협업 환경**을 제공하는 것이 본 프로젝트의 핵심 강점입니다.
